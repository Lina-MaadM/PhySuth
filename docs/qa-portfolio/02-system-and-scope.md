# 02 — ระบบ ระดับการทดสอบ และขอบเขต
---

## องค์ประกอบที่อยู่ในขอบเขตการทดสอบ

**1. Formula Map**  
หน้าเริ่มต้นของระบบ แสดงแผนภาพเครือข่ายของสูตร โดยโหนดแทนสูตรและเส้นเชื่อมแสดงตัวแปรร่วม

**2. Formula Catalog**  
หน้ารวบรวมสูตรทั้งหมดในรูปแบบรายการ

**3. Formula Detail**  
หน้าแสดงรายละเอียดของสูตร ประกอบด้วยข้อมูลของสูตร รายการตัวแปร และ Calculator สำหรับสูตรที่รองรับการคำนวณ

**4. Variable Index**  
หน้ารวบรวมตัวแปรทั้งหมดภายในระบบ แบ่งตามหัวข้อ (Topic)

**5. Relation View**  
หน้าแสดงสูตรทั้งหมดที่มีตัวแปรที่เลือกเป็นส่วนประกอบ

**6. Calculator**  
องค์ประกอบสำหรับคำนวณค่าตัวแปรของสูตรที่รองรับการคำนวณ และบันทึกค่าลงใน Variable Memory

**7. Variable Memory**  
องค์ประกอบสำหรับแสดงค่าตัวแปรที่บันทึกจากการคำนวณ

**8. History Bar**  
องค์ประกอบสำหรับบันทึกประวัติการเข้าชมสูตรและตัวแปร

**9. Session Overview**  
องค์ประกอบสำหรับสรุปประวัติการใช้งานภายใน Session ปัจจุบัน

## แนวทางการทดสอบ

ในเอกสารฉบับนี้ การทดสอบถูกแบ่งออกเป็น 3 ระดับ เพื่อใช้จัดกลุ่ม Test Case ตามวัตถุประสงค์ของการทดสอบ โดยนิยามดังกล่าวใช้เฉพาะภายในเอกสารนี้ 

| ชั้น | ความหมายในเอกสารนี้ | รายละเอียดอยู่ที่ |
|------|---------------------|-------------------|
| **Smoke** | ตรวจสอบว่าแต่ละหน้าและองค์ประกอบหลักสามารถเปิดใช้งานและแสดงผลได้ตามปกติ | [03 — Smoke / Navigation](./03-smoke-navigation.md) |
| **Navigation** | ตรวจสอบความถูกต้องของเส้นทางการนำทาง รวมถึงกรณีข้อมูลหรือพารามิเตอร์ไม่ถูกต้อง | [03 — Smoke / Navigation](./03-smoke-navigation.md) |
| **Functional** | ตรวจสอบพฤติกรรมและตรรกะการทำงานของฟีเจอร์ที่มีความซับซ้อน รวมถึงการจัดการข้อมูล การคำนวณ และการตอบสนองต่อข้อมูลที่ผิดปกติ | 
[04 — Functional Testing](./04-functional-testing.md) |


## ขอบเขตการทดสอบ

### อยู่ในขอบเขต

- การทดสอบแบบ Manual Testing
- ฟังก์ชันหลักของระบบตามที่ระบุในเอกสารฉบับนี้
- การบันทึกข้อบกพร่องที่พบระหว่างการทดสอบ

### อยู่นอกขอบเขตในรอบนี้

- การทดสอบบนอุปกรณ์เคลื่อนที่และ Responsive Layout
- Load, Performance และ Security Testing
- Automation Testing
- การทดสอบด้าน Usability ในห้องปฏิบัติการ (เช่น Eye-tracking)
